# Data Collection

## Developer Analytics:

Include analytics in SDK and Tools to gather information on developer usage, common workflows and identify common sticking points

## Surveys & Interviews:

Periodic surveys and requests for comment, combined with open “Office Hours” encourage developers to engage and feel heard

## Documentation Analytics:

Improve developer integration by including analytics in documentation and developer resources to identify search histories and navigation paths

Build for discovery of existing content with on ramps to deeper technical dives and openings for more specific documentation and tutorials guided by the developer community’s needs
